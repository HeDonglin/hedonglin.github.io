## 相关术语
 **git的历史记录**是不可修改的，也就是说你不能更改任何已经发生的事情。你做的任何操作都只是在原来的操作上修改。也就是说，即使你删除了一个分支，修改了一个提交，或者强制重置，你仍然可以回滚这些操作。
**切换**：初始创建一个分支，但git库还没有数据，叫切换；
**检出**：该分支之前已经提交过git库，那么检出就是经历了到仓库里取数据的过程；
**HEAD**：是指向当前所在的分支
**git查找配置信息顺序**先`etc\gitconfig`-->`Administrator\.gitconfig`-->`.git\config`;定义的值有冲突,那么后面的覆盖前面的;
**初始化后**最好要进行一次提交，避免使用了`git reset --hard`把文件给清除了；

## 配置快捷命令：
```
//转到用户名文件夹下
cd ~
//新建文件.bashrc
touch .bashrc
//快捷方式设置gs快捷命名是git status；alias gs='git status'
//例如alias gl='git log --oneline --decorate --graph' 当输入gl后面还可以输入其他参数，比如gl -3表示显示最近3条记录
//alias不加参数可以显示所有配置命令
```

## 出现问题后解决思路:
```
    是否添加了主机：git remote -v
    添加主机：git remote add origin git@github.com:xyhdl0929/xyhdl0929.github.io.git
    远程是否有了该分支：git push origin 本地分支：远程分支
    是否有追踪的分支：git branch -vv
    建立追踪分支：git branch --set-upstream 本地分支 origin/远程分支
    本地新建立一个分支追踪远程分支：git checkout -b <新的分支名> 远程主机名/分支名
    本地数据是否为最新：git pull
    设置本地默认主机：git push -u origin
```
